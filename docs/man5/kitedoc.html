DESCRIPTION

This man page defines the kitedoc(5) format for writing HTML documents. It is an "Extended HTML" format, i.e., it extends HTML with embedded macros. It is similar to manpage(5), but is used for arbitrary documents, with section number, tables of contents, and so forth. The basic syntax and capabilities of Extended HTML are defined in the ehtml(5) man page.

kitedoc(n) translates kitedoc(5) files into standard HTML files.

kitedoc(5) documents can contain automatically numbered and linked tables, figures, and sections; sections can be nested arbitrarily deeply, and the table of contents and lists of figures and tables can be generated automatically.

DOCUMENT STRUCTURE

kitedoc(5) can be used to write any kind of document that can be represented in HTML. If you wish to use kitedoc(5)'s automatic numbering and table-of-contents generation, your document will typically look like this:

  • The <document> header.
  • A list of change entries.
  • A table of <contents>.
  • Lists of <figures> and <tables>.
  • Zero or more <preface> sections. A preface section has a title but no section number, and cannot have subsections.
  • Zero or more numbered sections, each of which may have zero or more subsections.
  • Numbered sections may contain <table>s and <figure>s.
  • The </document> footer.

All of these features are optional; in particular, there's no need to have a list of figures or tables unless you have figures or tables.

IDENTIFIERS

To support auto-numbering and hyperlinking, every section, figure, and table has a unique identifier.

Section Identifiers

kitedoc(5) allows you to define automatically-numbered sections and subsections. Each section and subsection has an ID that uniquely identifies it; the ID also indicates how the section relates to its neighbors.

The ID for a top-level section (e.g., Section 1, Section 2, etc.) must be a single word containing no whitespace or punctuation.

The ID for a subsection is "parentId.name" where parentId is the parent's ID and name is a single word containing no whitespace or punctuation.

For example, a document might have an introductory section which contains two subsections: nomenclature to be used in the document, and a list of references. In the final document, we wish these sections to be numbered and titled as follows:

  • 1. Introduction
  • 1.1 Conventions used in this document
  • 1.2 References

In the kitedoc(5) input, the section headers are written as follows: 

<section intro "Introduction">
<section intro.conventions "Conventions used in this document">
<section intro.references "References">

In essence, each component of the numeric section number is replaced by a short word. You may nest sections six levels deep, but the normal constraints apply; for example, you can't add a section two levels below its parent, and the order of sections in the input needs to match the numeric order in the output.

Figure and Table Identifiers

Figure and table identifiers look like fig.name and tab.name. The prefix ensures that they won't be confused with section IDs.

Cross-References

kitedoc(n) supports ehtml(5)'s xref macro. Every section, table, and figure ID is an xref cross-reference ID. Documents can also link to the project's man pages, and to other project documents using a file path relative to the kitedoc(n) -docroot. Links to other projects are possible as well; see ehtml(5).

MACROS

kitedoc(5) man pages may use the following macros, as well as all commands defined in the ehtml(5) man page. Note that these macros are simply Tcl commands; you can define your own macros, as described in ehtml(5), and use them in your documents.

<banner>

Returns the banner for the top of the page. The banner consists of the project name, version, and description, as provided to kitedoc(n)'s format subcommand.

It should rarely be necessary to use this macro directly, as it is included automatically by <document>.

<contents>
This macro is replaced with a complete table of contents; the table of contents is automatically followed by lists of tables and figures if any tables and figures were defined.
<description>
Returns the description for the current project, as passed to kitedoc(n).
<document title>...</document>
These macros begin and end the document, and format the HTML header and footer, include the document's title and style sheet.
<figure id title filename>

Adds a figure to the document. The figure is identified by id, which must have the form fig.name; the title will be included in the caption. The figure itself is an image file called filename; the file must reside in the same directory as the generated HTML file, and must be a format recognized by the browser (e.g., GIF, PNG, or JPEG).

Figures will be auto-numbered, and included in the list of <figures>.

<figures>
This macro will be replaced by a list of the figures found in the document. Note that it is rarely necessary to use this macro explicitly, because the table of <contents> will include it automatically if there are any figures in the document.
<poc>
Returns the kitedoc(n) point-of-contact e-mail address as a "mailto" link.
<preface id title>

Adds an unnumbered section header to the document; it is uniquely identifed by id, and has the specified title: 

<preface foreword "Author's Foreword">

Unnumbered sections may not have subsections; hence, the id may not contain any periods.

Unnumbered sections may appear anywhere in the document, but really shouldn't be used anyway but the beginning or the end.

<project>
Returns the short name for the current project, as passed to kitedoc(n).
<section id title>
Adds a numbered section header to the document; the section has the specified id and title. See Section Identifiers for a discussion of how to choose the id.
<sectioncontents id>
Returns a table of contents for section id, that is, a list of its subsections, with links.
<standardstyle>

Returns the standard list of CSS styles, for inclusion in the HTML header.

It should rarely be necessary to use this macro directly, as it is included automatically by <document>.

<table id title>...</table>

Contains and captions a table object. The table id must have the form tab.name, while the title is arbitrary.

Tables will be auto-numbered, and included in the list of <tables>.

<textfigure id title>

Adds a "text figure" to the document, i.e., a figure consisting of an ehtml(5) <example> or <listing> rather than an image file. The relevant <example> or <listing> should follow immediately after the <textfigure> tag.

The figure is identified by id, which must have the form fig.name; the title will be included in the caption.

Text figures will be auto-numbered, and included in the list of <figures>.

<th>...</th>
Brackets a "pretty" table header.
<td>...</td>
Brackets a "pretty" table row.
<tables>
This macro will be replaced by a list of the tables found in the document. Note that it is rarely necessary to use this macro explicitly, because the table of <contents> will include it automatically if there are any tables in the document.
<version>
Returns the version number for the current project, as passed to kitedoc(n).
