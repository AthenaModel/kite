DESCRIPTION

This man page defines the project_macros(5) macro(n) macro set, which is provided by the kite docs command for use in both manpage(5) and kitedoc(5) documents.

This macro set provides access to project metadata, and also to the project code base.

MACROS

project_macros(5) provides the following project-specific macros.

<project subcomand ?args...?>

This macro gives access to project metadata.

<project name>
Returns the project's name.
<project version>
Returns the project's version number.
<project description>
Returns the project's description string.
<project tclsh script>

Executes the script in the development Tcl shell with access to the project's code base, and returns the result.

Documents will usually want to use the <toproject> and <fromproject> macros instead, however. This macro invokes the tclsh executable once each time it is called; the <toproject> and <fromproject> macros accumulate their scripts and invoke the tclsh a single time.

<project expand script>
Executes the script in the development Tcl shell with access to the project's code base, and then expands any macros in the result, returning the expansion.
<toproject script>

Executes the script in the development Tcl shell in the context of the project's code base, returning nothing. The script will be sent to the development Tcl shell as a batch with the scripts from all other <toproject> and <fromproject> macros in the same document.

This macro is usually used to require packages and do other setup.

<fromproject script>

Executes the script in the development Tcl shell in the context of the project's code base, returning the result. The script will be sent to the development Tcl shell as a batch with the scripts from all other <toproject> and <fromproject> macros in the same document.

This macro is usually used to call documentation generators in the project code base.

PROJECT TCL COMMANDS

Two special commands are made available to <project tclsh>, <project expand>, <toproject>, and <fromproject>:

get x

Returns its argument. This allows the script to easily return an interpolated string: 

<fromproject {get "This is a sum: [expr {1 + 1}]"}>
version

Returns the project version number, so that package require commands can be sure to require the correct version of the code. For example, a <toproject> command might look like this: 

<toproject {package require -exact mylib [version]}>
