SYNOPSIS

package require kiteutils 0.6.0
namespace import ::kiteutils::tclchecker
  • tclchecker check client script ?firstline?
  • tclchecker cmdsplit cmd ?num?
  • tclchecker getembedded string
  • tclchecker scriptsplit script ?firstline?

DESCRIPTION

tclchecker(n) defines a syntax checker for simple Tcl scripts of the kind run in a tclchecker(n) interpreter. In addition, it provides tools for breaking Tcl scripts into pieces, while tracking line numbers: individual commands, and individual words in a command.

COMMANDS

tclchecker check client script ?firstline?
Checks the script for obvious errors in the context of the client, which must be an instance of smartinterp(n), and returns a flat list of line numbers and error messages. If firstline is given, it is taken to be the line number of the first line of the script; it defaults to 1. Note that the result is not a dictionary; the same line number can appear multiple times.
tclchecker cmdsplit cmd ?num?

Splits a single Tcl command (as returned by scriptsplit) into individual words, returning a list of two items: a list of words, and a list of starting line numbers for each word. Thus, this command is usually used like this: 

lassign [tclchecker cmdsplit $command] words nums

If num is given, it's the starting line number for the command as a whole.

tclchecker getembedded string
Returns a list of the interpolated commands embedded in a string. textutil::expander does the heavy lifting.
tclchecker scriptsplit script ?firstline?

This command splits a script into individual commands, returning a dictionary of commands by initial line number. The caller can process the commands in sequence by iterating over the keys in the dictionary.

If firstline is given, it is used as the line number of the first line in the script. It defaults to 1.

Multiple commands on a single line are treated as one command.

This command does not recurse into control structure or proc bodies.

If the script cannot be split into commands because of an unterminated string, this command throws an error with error code UNTERMINATED.

ENVIRONMENT

Requires Tcl 8.6 or later.

To use this package in a Tcl script, the environment variable TCLLIBPATH must include the parent of the package directory.
