DESCRIPTION

macro(n) defines a Tcl-based macro expansion processor. Input consists of plain text with embedded macros; the macros are evaluated, and their return values are inserted into the body of text in their place.

See macro(5) for the basic syntax and standard macros.

Instances of macro(n) are wrapper around the Tcllib textutil::expander(n) module, using a smartinterp(n) object as the macro interpreter.

MACROS

The macro(5) man page describes some standard macros; and this module provides tools for adding additional macros. In general, new macros are defined in sets using the macroset(i) interface; a macroset(i) object is then registered with the macro(n) instance using the register method.

Macros are expanded in a safe smartinterp(n) interpreter. Any Tcl commands present in the interpreter can be used as a macro.

COMMANDS

macro name

Creates a new macro(n) object called name. The object is represented as a new Tcl command in the caller's scope; macro returns the fully-qualified form of the name.

The processor is manipulated via the subcommands of the macro command, which are as follows:

INSTANCE COMMAND

Each instance of the macro object has the following subcommands:

obj cget varname
obj cis cname
obj cname
obj cpop cname
obj cpush cname
obj cset varname value
obj cvar varname
obj errmode
obj lb
obj rb
obj setbrackets
obj where
These commands provide access to the textutil::expander(n); see that man page for details.
obj alias srcToken
obj alias srcToken {}
obj alias srcCmd targetCmd ?arg...?
obj alias srcCmd targetCmd ?arg...?
obj ensemble alias
obj proc name arglist body
obj smartalias alias min max argsyn prefix
These commands are used to define commands in the macro interpreter, i.e., to define new macros. See smartinterp(n) for details.
obj reset
Re-initializes the macro interpreter, defining any macros in registered macro sets. Any other macros defined by the client will need to be redefined.
obj register macroset
Registers a macroset(i) macro set with this macro processor. Its macros will be defined for use on the next reset.
obj eval script
Evaluates the script in the context of the macro interpreter.
obj expand text
Expands the text in two passes, and returns the result of the expansion. Use pass to write macros which are pass-sensitive.
obj expandfile name
Reads the contents of the text file called name, expands it in two passes, and returns the expansion. Use pass to write macros which are pass-sensitive.
obj expandonce text
Does a simple single-pass expansion of the text. This is useful in the implementation of macros that recursively expand one or more of their arguments, as it leaves the pass number unchanged.
obj pass
Returns the expansion pass number, 1 or 2. It's common to build up indices during the first pass and generate tables of links in the second.
obj warn text
Outputs the text to the console as a warning. TODO: There should be a -warncmd option to handle the warning.

SEE ALSO

macro(5), textutil::expander(n).

ENVIRONMENT

macro(n) requires Tcl 8.6 or later.

To use macro(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the kitedocs(n) library directory.
