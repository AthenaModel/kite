ehtml(n) -- Extended HTML Macro Set, submodule of kitedocs(n)

SYNOPSIS

package require kitedocs 0.6.0
namespace import ::kitedocs::*
  • ehtml css
  • ehtml install macro
  • ehtml configure opt val
  •     -docroot relpath
  •     -doctypes typelist

DESCRIPTION

ehtml(n) defines a macro(n) macro set that is a basis for "Extended HTML" documents. Extended HTML is a format similar to HTML text, but with an expanded set of tags defined using a macro processor. Many basic HTML tags work unchanged; new tags are defined for specific purposes. Tags not needed for purposes of documentation are not provided.

See ehtml(5) for the basic syntax and standard macros. See manpage(5) and kitedoc(5) for full document formats based on ehtml(5).

To use ehtml(n) directly, create an instance of macro(n) and register ehtml(n) with it: 

macro mp
mp register ::kitedocs::ehtml

MACROS

Extended HTML macros are simply Tcl commands whose return value is substituted into the input text. The ehtml(5) man page describes some standard macros

COMMANDS

ehtml css
Returns default CSS styles for the macros in this macro set. Clients are free to use them or not.
ehtml install macro
Installs the ehtml(5) macros into macro, an instance of macro(n). This is usually called by the macro(n) object itself.
ehtml configure opt val

Configures the macro set. The following options may be set:

-docroot relpath

Sets the relative path from the document file being processed to the project's docs/ directory. For docs/index.ehtml, the -docroot should be "."

The macro set's xref macro uses this to figure links to man pages and other documents. If the value of the option is "", then only explicit xrefset IDs are allowed.

-doctypes typelist

The typelist should be a list of file extensions, e.g., ".html .text .docx". The xref macro treats any xref ID ending with one of these extensions as a relative document path.

The default set includes the following:

  • .html
  • .htm
  • .txt
  • .text
  • .md
  • .docx
  • .xlsx
  • .pptx
  • .pdf

SEE ALSO

ehtml(5), manpage(5), kitedoc(5), macro(n)

ENVIRONMENT

ehtml(n) requires Tcl 8.6 or later.

To use ehtml(n) in a Tcl script, the environment variable TCLLIBPATH must include the parent of the kitedocs(n) library directory.
