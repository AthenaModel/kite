listx(n) -- List Utilities, submodule of kiteutils(n)

SYNOPSIS

package require kiteutils 0.5.0
namespace import ::kiteutils::*
  • got list
  • ladd listvar value
  • ldelete listvar value
  • lmaxlen list
  • lmerge listvar list
  • lpop stackvar
  • lpush stackvar item
  • lshift listvar
  • ltop stack
  • lzipper alist ?blist?

DESCRIPTION

listx(n) contains a set of list-manipulation commands.

COMMANDS

got list
Returns 1 if the list has at least one element, and 0 if it is empty.
ladd listvar value
Appends value to the list stored in listvar, only if it isn't already there. Updates listvar accordingly, and returns the (possibly updated) list.
ldelete listvar value
Removes value from the list stored in listvar if it is there. Updates listvar accordingly, and returns the (possibly updated) list.
lmaxlen list
Returns the length of the longest string in the list.
lmerge listvar list
Merges the elements of list into the list contained in variable listvar. The order of the resulting list is arbitrary, but it will contain each unique element of list at most once (unless the list in listvar already had duplicates).
lpop stackvar
Pops and returns the top item from a stack represented as a Tcl list. The top item in the stack is the item at the end of the list. Afterwards, the stackvar will contain the new stack. Returns the empty string if the stack is empty.
lpush stackvar item
Pushes one new item onto the top of the stack stored in stackvar. The stack is represented as a Tcl list; the top of the stack is the final item in the list, and so this command is equivalent to lappend. Returns the new stack.
lshift listvar
Given the name of a variable containing a list, removes the first element from the list and returns it, updating the variable. Effectively it pops the first element from the list.
ltop stack
Returns the top item from the stack, which is a stack represented as a Tcl list. The top item is the final item in the list. Returns the empty string if the stack is empty.
lzipper alist ?blist?

Returns a list formed by interleaving the elements of the two lists alist and blist. If one list is longer than the other, it is padded with empty items; and if blist is omitted, it's assumed to be empty.

Note that calling lzipper with one argument is a good way to create a dictionary with given keys and empty values.

ENVIRONMENT

Requires Tcl 8.6 or later.
